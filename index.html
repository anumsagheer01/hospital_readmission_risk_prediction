Predicting Hospital Readmission in Diabetic Patients

¶

What is Hospital Readmission?¶

Hospital readmission happens when a patient who was discharged from a hospital needs to be admitted again within a short period of time typically within 30 days. This often means that:

The patients condition was not fully treated
They didnt receive proper follow-up care
There were complications after discharge

For patients with diabetes, hospital readmissions are especially common. Diabetes affects multiple organs, and poor management can lead to severe health problems requiring emergency care.

Why Is This Important?

¶

High readmission rates put a strain on hospitals and healthcare workers
Patients experience more stress and financial burden
Hospitals lose money due to insurance penalties for excessive readmissions

If we can predict which patients are likely to be readmitted, hospitals can intervene early, provide better post-discharge care, and reduce unnecessary hospital visits.

# Define a function to categorize diagnosis codes
def categorize_diagnosis(code):
    if pd.isna(code):  # If the value is NaN, return "Unknown"
        return "Unknown"
    elif code.startswith('V') or code.startswith('E'):  # External causes & factors influencing health status
        return "External"
    else:
        try:
            num_code = float(code)  # Convert numeric codes
            if 1 <= num_code <= 139:
                return "Infectious"
            elif 140 <= num_code <= 239:
                return "Neoplasms"
            elif 240 <= num_code <= 279:
                return "Endocrine"
            elif 280 <= num_code <= 289:
                return "Blood_Disease"
            elif 290 <= num_code <= 319:
                return "Mental_Disorders"
            elif 320 <= num_code <= 389:
                return "Nervous_System"
            elif 390 <= num_code <= 459:
                return "Circulatory_System"
            elif 460 <= num_code <= 519:
                return "Respiratory_System"
            elif 520 <= num_code <= 579:
                return "Digestive_System"
            elif 580 <= num_code <= 629:
                return "Genitourinary_System"
            elif 630 <= num_code <= 679:
                return "Pregnancy_Complications"
            elif 680 <= num_code <= 709:
                return "Skin_Disease"
            elif 710 <= num_code <= 739:
                return "Musculoskeletal"
            elif 740 <= num_code <= 759:
                return "Congenital_Anomalies"
            elif 760 <= num_code <= 779:
                return "Perinatal_Conditions"
            elif 780 <= num_code <= 799:
                return "Symptoms_Signs"
            elif 800 <= num_code <= 999:
                return "Injury_Poisoning"
            else:
                return "Other"
        except ValueError:
            return "Other"

Conclusion¶

In this project, I developed a machine learning model to predict hospital readmission within 30 days for diabetic patients. Using a dataset of 101,766 patient records from 130 US hospitals, I performed data preprocessing, feature engineering, and model training to identify high-risk patients and improve hospital resource allocation.

Key Steps and Results:¶

  1. Data Processing

    • Cleaned and preprocessed 50+ features, handling missing values and encoding categorical variables.
    • Scaled numerical features such as time in hospital, number of medications, and lab procedures for better model performance.

  2. Feature Selection & Importance

    • Identified the top predictors of readmission using SHAP analysis and feature importance methods.
    • The most influential features were:
      • Number of in patient visits (strongest predictor)
      • Number of diagnoses
      • Time spent in hospital
      • Number of emergency visits
      • Number of lab procedures
      • Use of insulin and diabetes medications
      • Payer code and medical specialty (potentially reflecting socio-economic and treatment factors)

  3. Model Training & Evaluation

    • Tested multiple models, including Logistic Regression and Random Forest.
    • The optimized Random Forest model achieved:
      • 89.27% accuracy
      • 98.94% recall (high ability to detect actual readmissions)
      • AUC-ROC score of 0.929, meaning excellent distinction between readmitted and non-readmitted patients.

  4. Deployment

    • Built a Flask API to make the model accessible for real-time hospital predictions.
    • Deployed the API using Docker, ensuring scalability and ease of integration into healthcare systems.

This project provides a data-driven solution for hospitals to proactively identify high-risk diabetic patients, improve post-discharge care, and reduce unnecessary readmissions, ultimately enhancing patient outcomes and optimizing hospital resources.
